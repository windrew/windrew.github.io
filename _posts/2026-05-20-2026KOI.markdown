## 초등부

### 이웃 (2교시 1번)
모든 학생 $i$에 대해서 다른 학생 $j$에 대해 이웃인지 아닌지를 문제에서 주어진 조건을 구현하면 확인할 수 있습니다.

구체적으로, $a[i] = a[j], |i-j| \le K_1$이거나 $a[i] \ne a[j], |i-j| \le K_1$인지를 체크하되, 지문에 나와있는 대로 $i = j$인 경우를 세지 않게 유의하면 됩니다.

```
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;

ll a[3005];

int main()
{
    ll i,j,n,k1,k2;
    scanf("%lld %lld %lld",&n,&k1,&k2);
    for(i=1;i<=n;i++)
    {
        scanf("%lld",&a[i]);
    }
    for(i=1;i<=n;i++)
    {
        ll nans = 0;
        for(j=1;j<=n;j++)
        {
            if(i == j)continue;
            if(a[i] == a[j] && abs(j - i) <= k1)nans++;
            if(a[i] != a[j] && abs(j - i) <= k2)nans++;
        }
        printf("%lld ",nans);
    }
}

```

### 가위바위보 (2교시 2번)
각 위치 $i$의 손모양에 대해 $1$부터 $i-1$ 사이에 $i$를 이기는 손모양만 존재하면 어떻게 하여도 그 손모양들을 없앨 방법이 없습니다.

마찬가지로, 오른쪽에 대해서도 $i+1$부터 $N$ 사이에 $i$를 이기는 손모양만 존재하면 그러한 손모양을 없앨 방법이 없습니다.

아닌 경우에 대해서는 이기는 손모양들에 대해 $i$가 아닌 인접한 손모양과 계속해서 가위바위보를 시키면 $i$의 왼쪽, 오른쪽에 마지막에 남는 손모양은 $i$를 이기지 못하는 손모양임을 알 수 있습니다. 따라서, 위 두 가지 경우에 속하는지 아닌지를 체크하면 충분합니다.

```
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;

char s[200005];
ll lr,lp,ls,rr,rp,rs;

int main()
{
    ll i,j,n;
    scanf("%lld",&n);
    scanf("%s",s);
    for(i=0;i<n;i++)
    {
        if(s[i] == 'R')rr++;
        else if(s[i] == 'S')rs++;
        else rp++;
    }
    for(i=0;i<n;i++)
    {
        if(s[i] == 'R')rr--;
        else if(s[i] == 'S')rs--;
        else rp--;
        if(s[i] == 'R')
        {
            if((rp && !rs) || (lp && !ls))printf("0");
            else printf("1");
        }
        else if(s[i] == 'S')
        {
            if((rr && !rp) || (lr && !lp))printf("0");
            else printf("1");
        }
        else
        {
            if((rs && !rr) || (ls && !lr))printf("0");
            else printf("1");
        }
        if(s[i] == 'R')lr++;
        else if(s[i] == 'S')ls++;
        else lp++;
    }
}
```

## 중등부

### 수열 정렬하기 (2교시 2번)

서브태스크 5의 풀이를 생각해봅시다. 편의상 $A$를 순열이라고 생각하겠습니다. 그렇게 생각해도 무방합니다.

$i$와 $i+1$의 순서에 집중합시다. $i$보다 작은 수에 시행을 하거나 $i$보다 큰 수의 시행을 하여도 순서가 바뀌지 않습니다.

그러나, $i$에 연산을 한다면, $i+1$과 $i$의 순서가 역전되어 있었다면 바뀌게 됩니다. 이때, $i$와 $i+1$의 순서가 역전되어 있는 상황은 오름차순으로 정렬되지 못한 상황이므로 순서가 역전되어 있는 $i$에는 무조건 연산을 해주어야 한다는 것을 알 수 있습니다.

해당하는 값들에 전부 연산을 해준다면 모든 $i$에 대해 $i$ 이후에 $i+1$이 등장합니다. 이러한 조건을 만족하는 경우가 오름차순으로 정렬된 상태밖에 없다는 것은 쉽게 알 수 있습니다. 따라서, 해당하는 $i$의 개수를 세주면 서브태스크 5를 풀 수 있습니다.

풀 태스크 또한 크게 다르지 않습니다. 해당 풀이에서 순서를 체크할 때, 가장 왼쪽의 $i+1$과 가장 오른쪽의 $i$의 순서가 역전되어 있다면 서브태스크 5와 같이 무조건 $i$에 연산이 필요하고, 그렇지 않다면 연산을 하지 않아도 됩니다. 따라서 그러한 $i$의 개수를 적절히 빠르게 세주면 됩니다.


```
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;

ll v[300005],c[300005];

struct A
{
    ll v,idx,nv;
}d[300005];

bool cmp(A i,A j)
{
    return i.v < j.v;
}

bool cmp0(A i,A j)
{
    return i.idx < j.idx;
}

int main()
{
    ll i,j,n,ans = 0;
    scanf("%lld",&n);
    for(i=1;i<=n;i++)
    {
        scanf("%lld",&d[i].v);
        d[i].idx = i;
    }
    sort(d+1,d+1+n,cmp);
    ll cnt = 1;
    for(i=1;i<=n;i++)
    {
        if(d[i-1].v == d[i].v)d[i].nv = d[i-1].nv;
        else d[i].nv = cnt++;
    }
    sort(d+1,d+1+n,cmp0);
    for(i=1;i<=n;i++)
    {
        if(v[d[i].nv+1] && !c[d[i].nv])
        {
            c[d[i].nv] = 1;
            ans++;
        }
        v[d[i].nv] = 1;
    }
    printf("%lld",ans);
}
```

## 고등부

### 친구 (2교시 1번)

기본적으로 초등부 1번의 이웃과 상당히 유사한 문제입니다. 대신, $N$제한이 더 크기 때문에 더 빠른 풀이를 요구합니다.

각 학생 $i$에 대해 문제에서 요구하는 값은 두 학생이 서로 같은 학교에 다니고, 두 학생의 집 사이의 거리가 $K_1$이하이거나 두 학생이 서로 다른 학교에 다니고, 두 학생의 집 사이의 거리가 $K_2$이하이게 하는 다른 학생 $j$의 수 입니다.

이는 어떤 학교든 상관없이 모든 학교에 대해 집 사이의 거리가 $K_2$이하이게 하는 학생의 수 - 같은 학교에 다니면서 집 사이의 거리가 $K_2$이하이게 하는 학생의 수 + 같은 학교에 다니면서 집 사이의 거리가 $K_1$이하인 학생의 수로 나타낼 수 있습니다.

따라서, 각 학교에 다니는 학생들의 위치를 관리하는 vector와 전체 학생들의 위치를 관리하는 vector를 정렬한 이후, 이분 탐색을 통해서 해당하는 학생의 수를 잘 계산해주면 $O(nlogn)$에 전체 문제를 해결할 수 있습니다.

```
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;

ll x[500005],s[500005];
vector < ll > v[500005];
vector < ll > all;

int main()
{
    ll i,j,n,k1,k2;
    scanf("%lld %lld %lld",&n,&k1,&k2);
    for(i=1;i<=n;i++)
    {
        scanf("%lld %lld",&x[i],&s[i]);
        v[s[i]].push_back(x[i]);
        all.push_back(x[i]);
    }
    sort(all.begin(),all.end());
    for(i=1;i<=n;i++)
    {
        sort(v[i].begin(),v[i].end());
    }
    for(i=1;i<=n;i++)
    {
        ll idx1 = lower_bound(all.begin(),all.end(),x[i]-k2) - all.begin();
        ll idx2 = upper_bound(all.begin(),all.end(),x[i]+k2) - all.begin();
        ll ans = idx2 - idx1;
        idx1 = lower_bound(v[s[i]].begin(),v[s[i]].end(),x[i]-k1) - v[s[i]].begin();
        idx2 = upper_bound(v[s[i]].begin(),v[s[i]].end(),x[i]+k1) - v[s[i]].begin();
        ans += idx2 - idx1;
        idx1 = lower_bound(v[s[i]].begin(),v[s[i]].end(),x[i]-k2) - v[s[i]].begin();
        idx2 = upper_bound(v[s[i]].begin(),v[s[i]].end(),x[i]+k2) - v[s[i]].begin();
        ans -= idx2 - idx1;
        printf("%lld ",ans - 1);
    }
}
```

### 산책로 (2교시 2번)

풀 태스크를 풀기 위해 트리 위에서 dp를 해보는 것을 생각할 수 있습니다. 두 정점 사이를 지나는 경로에 대한 문제는 어떤 정점을 지나는 경로에 대한 답을 구하는 분할정복 문제로 생각하는 센트로이드 분할 문제로 생각할 수 있습니다.

센트로이드 분할을 알고 있다는 전제 하에, 센트로이드 분할을 사용하기로 하였다면 크게 어렵지 않게 풀이를 떠올릴 수 있습니다.

어떤 정점을 지나는 경로 중 $S - M$의 최댓값은 각 정점 $i$에 대해 센트로이드와 $i$까지의 $M$을 구하였다면, 이 $M$보다 작거나 같으면서 최대 $S$값을 가지는 다른 정점 $j$와의 연결만 고려하면 구할 수 있습니다.

따라서 $M$에 대해 좌표압축을 수행한 세그먼트 트리를 통해서 해결할 수 있고, 이는 $O(Nlog^2N)$의 시간복잡도를 가지므로 상수가 작은 펜윅 트리를 활용해야 AC를 받을 수 있습니다.

```
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;
#define siz 524288
#define inf 4000000000000000000
#pragma GCC optimize("O3")
#pragma GCC optimize("Ofast")
#pragma GCC optimize("unroll-loops")

vector < pair < ll, ll > > adj[300005];
vector < array < ll, 4 > > now;
ll a[300005],c[300005],l[300005],sz[300005],idx[300005],v[300005];
ll ans = -inf,ans1,ans2;
vector < ll > clr;

struct A
{
    ll v,idx;
}d[300005];

bool cmp(A i,A j)
{
    return i.v < j.v;
}

bool cmp0(A i,A j)
{
    return i.idx < j.idx;
}

void dfs(ll node,ll par)
{
    sz[node] = 1;
    ll i;
    for(i=0;i<adj[node].size();i++)
    {
        if(adj[node][i].first == par)continue;
        if(v[adj[node][i].first])continue;
        dfs(adj[node][i].first,node);
        sz[node] += sz[adj[node][i].first];
    }
}

ll centroid(ll node,ll par,ll half)
{
    ll i;
    for(i=0;i<adj[node].size();i++)
    {
        if(adj[node][i].first == par)continue;
        if(v[adj[node][i].first])continue;
        if(sz[adj[node][i].first] >= half)return centroid(adj[node][i].first,node,half);
    }
    return node;
}

void dfs1(ll node,ll par,ll nm,ll ns,ll nidx)
{
    ll i;
    nm = max(nm,a[node]);
    nidx = max(nidx,idx[node]);
    now.push_back({node,nm,ns,nidx});
    for(i=0;i<adj[node].size();i++)
    {
        if(adj[node][i].first == par)continue;
        if(v[adj[node][i].first])continue;
        dfs1(adj[node][i].first,node,nm,adj[node][i].second + ns,nidx);
    }
}

struct Node
{
    ll v,idx;
}tree[siz+5], pt[siz+5];

void update(ll s,ll e,ll l,ll r,ll node,ll v,ll idx)
{
    if(v == 0 && idx == 0)
    {
        pt[l] = {0, 0};
        for(ll i=l; i<=siz; i+=(i&-i)) tree[i] = {0, 0};
        return;
    }
    pt[l] = {v, idx};
    for(ll i=l; i<=siz; i+=(i&-i))
    {
        if(tree[i].v < v) tree[i] = {v, idx};
    }
}

Node query(ll s,ll e,ll l,ll r,ll node)
{
    if(l == r) return pt[l];
    Node ret = {0, 0};
    for(ll i=r; i>0; i-=(i&-i))
    {
        if(ret.v < tree[i].v) ret = tree[i];
    }
    return ret;
}

void dq(ll node)
{
    dfs(node,-1);
    ll c = centroid(node,-1,sz[node]/2);
    v[c] = 1;
    ll i,j;
    for(i=0;i<adj[c].size();i++)
    {
        if(v[adj[c][i].first])continue;
        dfs1(adj[c][i].first,-1,a[c],adj[c][i].second,idx[c]);
        for(j=0;j<now.size();j++)
        {
            Node ret = query(1,siz,1,now[j][3],1);
            ll nv = ret.v + now[j][2] - now[j][1];
            if(nv > ans)
            {
                ans = nv;
                ans1 = now[j][0];
                ans2 = ret.idx;
                if(ans2 == 0)ans2 = c;
            }
        }
        for(j=0;j<now.size();j++)
        {
            clr.push_back(now[j][3]);
            Node ret = query(1,siz,now[j][3],now[j][3],1);
            if(ret.v < now[j][2])update(1,siz,now[j][3],now[j][3],1,now[j][2],now[j][0]);
        }
        now.clear();
    }
    for(i=0;i<clr.size();i++)
    {
        update(1,siz,clr[i],clr[i],1,0,0);
    }
    for(i=(ll)adj[c].size()-1;i>=0;i--)
    {
        if(v[adj[c][i].first])continue;
        dfs1(adj[c][i].first,-1,a[c],adj[c][i].second,idx[c]);
        for(j=0;j<now.size();j++)
        {
            Node ret = query(1,siz,1,now[j][3],1);
            ll nv = ret.v + now[j][2] - now[j][1];
            if(nv > ans)
            {
                ans = nv;
                ans1 = now[j][0];
                ans2 = ret.idx;
                if(ans2 == 0)ans2 = c;
            }
        }
        for(j=0;j<now.size();j++)
        {
            Node ret = query(1,siz,now[j][3],now[j][3],1);
            if(ret.v < now[j][2])update(1,siz,now[j][3],now[j][3],1,now[j][2],now[j][0]);
        }
        now.clear();
    }
    for(i=0;i<clr.size();i++)
    {
        update(1,siz,clr[i],clr[i],1,0,0);
    }
    clr.clear();
    for(i=0;i<adj[c].size();i++)
    {
        if(v[adj[c][i].first])continue;
        dq(adj[c][i].first);
    }
}

int main()
{
    ll i,j,n,k1,k2;
    scanf("%lld",&n);
    for(i=1;i<=n;i++)
    {
        cin >> a[i];
        d[i].v = a[i];
        d[i].idx = i;
    }
    sort(d+1,d+1+n,cmp);
    for(i=1;i<=n;i++)
    {
        d[i].v = i;
    }
    sort(d+1,d+1+n,cmp0);
    for(i=1;i<=n;i++)
    {
        idx[i] = d[i].v;
    }
    for(i=1;i<n;i++)
    {
        cin >> c[i];
    }
    for(i=1;i<n;i++)
    {
        cin >> l[i];
        adj[i].push_back(make_pair(c[i],l[i]));
        adj[c[i]].push_back(make_pair(i,l[i]));
    }
    dq(1);
    cout << ans1 << " " << ans2;
}
```

### 점프 (2교시 3번)

절댓값을 떼고, 위와 아래를 분리해서 생각합시다. 내 위에 있는 발판으로의 이동은 이동할 때 위로만 이동하고, 내 밑에 있는 발판으로의 이동은 이동할 때 내 밑으로만 이동해서(즉, y좌표가 오름차순, 내림차순이 되도록) 도달하는 경로가 무조건 존재함을 증명할 수 있습니다.

이때, 위로 이동하는 경우만 고려합시다. (아래로 이동하는 경우도 비슷하게 할 수 있으므로) 나와 가장 가까운 내 위의 발판으로 이동한다면 그 발판 이상의 높이의 발판으로는 여전히 도달할 수 있습니다. 그리고, 그 발판과 나 사이의 발판들에 대해서는 그 발판을 거쳐서는 이동할 수 없습니다.

따라서 그 발판에서 오름차순으로 이동하여 도달할 수 있는 발판의 수 + 1 + 그 발판과 나 사이의 발판 개수가 현재 나한테서 오름차순으로 이동할 수 있는 발판의 수임을 알 수 있습니다.

가장 가까운 내 위/아래의 발판 위치와 그 사이의 발판 개수는 세그먼트 트리를 활용하면 $O(logn)$에 구할 수 있습니다. 따라서 전체 문제를 $O(nlogn)$에 해결할 수 있습니다.

```
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;
#define siz 524288

ll y[300005],l[300005],r[300005],ldp[300005],rdp[300005],mdp[300005];
map < ll, ll > mp;

struct Node
{
    ll v,idx = 300001;
}tree1[siz*4],tree2[siz*4];

void update1(ll s,ll e,ll l,ll r,ll node,ll v)
{
    if(s > r || e < l)return;
    if(l <= s && e <= r)
    {
        tree1[node].v++;
        tree1[node].idx = v;
        return;
    }
    ll mid = (s+e)/2;
    update1(s,mid,l,r,node*2,v);
    update1(mid+1,e,l,r,node*2+1,v);
    tree1[node].v = tree1[node*2].v + tree1[node*2+1].v;
    tree1[node].idx = min(tree1[node*2].idx,tree1[node*2+1].idx);
}

Node query1(ll s,ll e,ll l,ll r,ll node)
{
    if(s > r || e < l)return tree1[siz*4-1];
    if(l <= s && e <= r)return tree1[node];
    ll mid = (s+e)/2;
    Node a = query1(s,mid,l,r,node*2);
    Node b = query1(mid+1,e,l,r,node*2+1);
    Node ret;
    ret.v = a.v + b.v;
    ret.idx = min(a.idx,b.idx);
    return ret;
}

        mdp[i] = query1(1,siz,y[i],y[i],1).v + 1;
        ret = query1(1,siz,y[i],r[i],1);
```
